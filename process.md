Before writing code, you should be able to answer:

What are all possible states?
What can go wrong?
What data survives between loops?
What is temporary?
What should be stored?
What should happen after failure?

That is programming.

### Do this The 5-Minute Flow Method

Before coding, write ONLY this:

1. What is the input?

Example:

amount
category
description

2. What can go wrong?

Example:

non-number amount
empty category
invalid date

3. What data must persist?

Example:

total spent
transaction history

4. What repeats?

Example:

adding transactions

5. When does the program stop?

Example:

user chooses exit
